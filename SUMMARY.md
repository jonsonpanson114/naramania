# プロジェクト引き継ぎドキュメント: Naramania

## 1. プロジェクト概要
### 目的
奈良県および県内各自治体の入札情報（PPI）を自動収集し、さらにPDF（公告・結果）からAIを用いて「予定価格」「落札者」「業務概要」などのインテリジェンス情報を抽出・可視化するポータルサイト。

### 主な機能
- **マルチ自治体スクレイピング**: 奈良県、奈良市、橿原市を含む県内23自治体の入札情報を個別ロジックで取得。
- **AIインテリジェンス抽出**: Gemini API を用い、複雑なPDFファイルから非構造化データを抽出し JSON 化。
- **自動運用**: GitHub Actions により、毎日午前9時（日本時間）に全自治体のデータを自動更新。
- **分析ダッシュボード**: 抽出した予定価格や落札傾向をチャート形式で可視化。

---

## 2. 技術スタックと構成
### 主要技術
- **Frontend/Backend**: Next.js (App Router), TypeScript
- **Scraping**: Playwright, Axios, Cheerio
- **AI**: Google Gemini API
- **Workflow**: GitHub Actions
- **Styling**: Vanilla CSS (Next.js CSS Modules)

### ディレクトリ構成
- `src/scrapers/`: 各自治体（23種）のスクレイピングロジック。`index.ts` が司令塔。
- `src/app/`: Next.js のページ（ダッシュボード、案件詳細）および API ルート。
- `scripts/`: データクリーニング、個別分析、デバッグ用のユーティリティ群。
- `scripts/output/`: 実行ログ（v11, v12, v14 など）。
- `scraper_result.json`: 全自治体の集計済みマスターデータ。

---

## 3. 実装済みの内容
- **最新AIモデルへの換装**: **Gemini 3.1 Flash Lite Preview** (2026-03-03リリース) を導入。PDFからのデータ抽出精度と速度が劇的に向上。
- **インテリジェンス抽出の自動化と堅牢化**: 
  - 1件ごとの逐次保存機能（進捗損失防止）。
  - 高度なリトライロジック（429 Too Many Requests, 503 Service Unavailable, 500対応）。
- **データトレーサビリティ**: `extractionSource` フィールドを導入し、データがスクレイパー由来か、最新AI（gemini_3.1）由来かを明示。
- **マルチ自治体対応**: 橿原市、葛城市、大和高田市などの複雑なPDF（開札録・公告）からの情報抽出に成功。
